3. Information We Handle

The App does not collect or transmit personal information to our servers. The information handled by the App is as follows.

3.1 Information Processed Locally

The following information is processed exclusively on the user's device and is not sent to our servers.

  • Window information: Application name, window title, window position and size (for shake detection and context capture)
  • File paths: Folder paths and file URLs displayed in Finder windows
  • Memory data: Knowledge extracted and stored by the AI agent (saved in a local SQLite database)
  • Session history: Conversation history with the agent (saved as local files)
  • Configuration: Rules, skills, window links, and other user settings

3.2 Information Sent to Third-Party Services

To execute the App's features, the following information is sent from the user's device to third-party services.

  • To Claude Code (Anthropic): User-entered prompts, task-related file contents, and folder structure. This information is required for AI agent execution.
  • To Gmail (Google): When using the Mail Check feature, email data is retrieved through the user's authenticated Google account. Retrieved data is processed locally and is not sent to our servers.

3.3 Automatically Transmitted Information

  • Update checks: The auto-update feature (Sparkle) sends the App's version and OS version to GitHub Pages to check for available updates.

4. Purpose of Use

Information handled by the App is used exclusively for the following purposes:

  1. AI agent task execution (file operations, code generation, etc.)
  2. Window context capture and shake gesture detection
  3. Email retrieval, triage, and summarization (Mail Check feature)
  4. Knowledge storage and recall (Memory feature)
  5. Automatic app updates

5. Disclosure to Third Parties

We do not provide user information to third parties, except in the following cases:

  1. When information is sent to third-party services (Claude Code, Google, etc.) based on user actions as a feature of the App (see Section 3.2)
  2. When disclosure is required by law
  3. When necessary to protect the life, body, or property of an individual, and obtaining consent is impractical

8. Data Storage and Management

  1. All data stored by the App resides exclusively on the user's local device.
  2. We do not operate servers that hold user data.
  3. Management of locally stored data (backup, deletion, etc.) is the user's responsibility.

9. User Rights

Users can manage their information in the following ways:

  • Data deletion: Uninstalling the App removes all locally stored data created by the App.
  • Permission revocation: System permissions granted to the App can be revoked at any time from macOS System Settings.
  • Mail feature opt-out: Disconnecting the Google account disables the Mail Check feature.
  • Auto-update opt-out: Automatic update checks can be disabled in the App's settings.

10. Security Measures

  1. The App minimizes the risk of data exposure by storing user data exclusively on the local device, with no transmission to our servers.
  2. Communications with third-party services are encrypted using TLS (Transport Layer Security).
  3. Google account integration uses OAuth 2.0 authentication. The App never stores your password.

11. Changes to This Policy

  1. We may update this Policy at any time, including in connection with App updates that add or change features. The types of information handled and their purposes of use may change as the service evolves.
  2. The revised Policy takes effect upon posting on the App's website.
  3. If we begin collecting new categories of personal information, or materially change the purpose of use of existing information, we will notify users in advance through in-app notification or an announcement on the website.
  4. Continued use of the App after any update constitutes acceptance of the revised Policy.
